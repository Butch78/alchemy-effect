Effect AI's [`Persistence`](https://effect-ts.github.io/effect/docs/unstable/persistence)
module is how the chat primitives keep a conversation's message history
around between calls. By default the chat store lives in memory, which
is fine for in-process demos but useless across a Worker fleet.

`Cloudflare.DurableObjectChatPersistence` is a `BackingPersistence`
layer that uses the surrounding Durable Object's `state.storage` as
the store, so each DO instance keeps its own conversation history,
namespaced by `storeId`.

## Wire the layer into a chat-backing DO

The layer pulls `DurableObjectState` out of context, so it can only be
provided inside a DO. Wire it as part of the inner Effect:

```typescript
// src/chat-backend.ts
import * as Cloudflare from "alchemy/Cloudflare";
import * as Effect from "effect/Effect";
import * as Layer from "effect/Layer";
import { Chat, LanguageModel } from "effect/unstable/ai";
import { Persistence } from "effect/unstable/persistence";

export default class ChatBackend extends Cloudflare.DurableObjectNamespace<ChatBackend>()(
  "ChatBackend",
  Effect.gen(function* () {
    return Effect.gen(function* () {
      const persistence = yield* Persistence.layerResultPersisted({
        storeId: "alchemy.chat",
      }).pipe(Layer.provide(Cloudflare.DurableObjectChatPersistence));

      return {
        send: (threadId: string, prompt: string) =>
          Effect.gen(function* () {
            const chat = yield* Chat.fromPrompt({ prompt });
            const response = yield* LanguageModel.generateText({ prompt });
            return response.text;
          }).pipe(Effect.provide(persistence)),
      };
    });
  }),
) {}
```

`Persistence.layerResultPersisted({ storeId })` is the seam Effect AI
exposes for cached/replayable AI calls. By layering
`DurableObjectChatPersistence` underneath, every call's input/output
record is stored in the DO's `state.storage` under the
`alchemy.chat:` prefix. Restart, hibernation, even a redeploy — the
records survive.

## Multiple stores in the same DO

Pass different `storeId`s for different concerns. Each store's keys
are namespaced with `${storeId}:`, so they don't collide:

```typescript
const aiPersistence = yield* Persistence.layerResultPersisted({
  storeId: "alchemy.chat",
}).pipe(Layer.provide(Cloudflare.DurableObjectChatPersistence));

const auditPersistence = yield* Persistence.layerResultPersisted({
  storeId: "alchemy.audit",
}).pipe(Layer.provide(Cloudflare.DurableObjectChatPersistence));
```

## Storage shape

Behind the `BackingPersistence` interface, each entry is stored as a
single struct value via `state.storage.put(key, value)`. Cloudflare DO
storage is SQLite-backed, so reads are O(log n) and bulk operations
(`getMany`, `setMany`) translate to one SQL query each.

:::caution[No TTL]
DO storage has no native TTL — the persistence layer ignores TTLs you
pass to `set`. If you need expiry, layer a TTL-aware backing store on
top, or run a periodic `clear`.
:::

## Recap

- `DurableObjectChatPersistence` is a `BackingPersistence` layer
  backed by `DurableObjectState.storage`, providing a per-instance
  store for Effect AI's chat persistence machinery.
- Provide it underneath
  `Persistence.layerResultPersisted({ storeId })` inside the DO's
  inner Effect; multiple `storeId`s coexist in one DO.
- Survives restarts, hibernation, and redeploys; storage is per-DO
  instance, so the natural unit of isolation is "one chat per
  `getByName(id)`".
