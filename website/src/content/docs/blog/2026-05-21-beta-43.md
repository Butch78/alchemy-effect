## PlanetScale, branched per PR

`alchemy/Planetscale` is the new provider — `MySQLDatabase` and
`PostgresDatabase`, branches, credentials, and a Drizzle
integration that plugs straight into Cloudflare Hyperdrive. The
cluster is owned by one stage; PR previews fork off it with
`Resource.ref(id, { stage })`.

```typescript
const database = stage.startsWith("pr-")
  ? yield* Planetscale.PostgresDatabase.ref("app-db", {
      stage: `staging-${stage}`,
    })
  : yield* Planetscale.PostgresDatabase("app-db", {
      region: { slug: "us-east" },
      clusterSize: "PS_10",
    });
```

## Service Layers — refreshed docs + new guide

A **Service Layer** packages infrastructure behind a typed
service. The Layer owns the resources and bindings; consumers
depend only on the service interface and can't tell which cloud
is underneath:

```typescript
export const JobServiceKV = Layer.effect(
  JobService,
  Effect.gen(function* () {
    const Namespace = yield* Cloudflare.KVNamespace("Jobs");
    const kv = yield* Cloudflare.KVNamespaceBinding.bind(Namespace);
    return {
      getJob: Effect.fn(function* (id: string) {
        return yield* kv.get<Job>(id, "json");
      }),
    };
  }),
);
```

[Service Layers](/concepts/layers) was rewritten around a
Worker-shaped encapsulation example instead of an abstract
`getJob(id)` helper, and there's a new step-by-step guide for
building one yourself.
